This Supabase backend powers a SaaS expense tracker with user authentication, category management, and expense CRUD operations. Tables: `app_a401_profiles` (user profiles auto-created on signup), `app_a401_categories` (default 8 categories seeded per user, customizable), and `app_a401_expenses` (expense entries with amount, description, date, and category FK). All tables are RLS-protected and scoped by `auth.uid()`. Realtime is enabled on `app_a401_expenses` and `app_a401_categories` for live updates across sessions. A trigger auto-creates user profiles and default categories on signup. Indexes optimize queries by user, date, and category. CSV export is handled client-side in the frontend. No edge functions are required—all CRUD happens via Supabase auto-generated REST API with RLS enforcement.
